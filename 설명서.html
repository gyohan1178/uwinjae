1
파일 구성
시스템은 두 개의 HTML 파일로 구성됩니다
📄
index.html
학생들이 서류를 제출하는 접수 페이지입니다. 링크를 공개하여 학생들에게 배포합니다.
공개 배포용
🔒
admin.html
담당자가 접수 현황을 확인하고 관리하는 대시보드입니다. 외부에 공유하지 마세요.
담당자 전용
현재 데이터 저장 방식 안내
현재는 브라우저 로컬 저장소에 데이터가 저장됩니다. 다른 기기에서는 데이터가 공유되지 않습니다. 실 서비스를 위해서는 Firebase 연동이 필요합니다.
2
학생 접수 방법
index.html 기준 안내
신규 접수 신규 접수 탭
1
분야 선택
현재 접수 가능한 분야만 활성화되어 표시됩니다. 접수 기간이 아닌 분야는 자동으로 잠깁니다.
2
기본 정보 입력
이름, 생년월일, 연락처, 학교, 주소를 입력합니다. 주소는 주소 검색 버튼으로 도로명주소를 찾아 입력합니다. 고등부·특기 분야는 생년월일 생략 가능합니다.
3
서류 파일 업로드
서류 항목별로 PDF 파일을 개별 업로드합니다. 파일당 최대 10MB. 여러 서류를 하나로 합쳐서 올리면 안 됩니다.
4
접수 완료 및 접수번호 저장
접수하기 버튼 클릭 후 발급되는 접수번호를 반드시 저장합니다. 보완 서류 제출 시 필요합니다.
보완 서류 제출 보완 서류 제출 탭
1
접수번호 입력 후 조회
담당자에게 받은 누락 안내 후, 처음 발급받은 접수번호를 입력합니다.
2
누락 서류만 표시
미제출 서류만 목록으로 표시됩니다. 해당 서류 PDF 파일만 업로드 후 제출합니다.
서류 제출 시 주의사항
• 반드시 PDF 파일로 제출 (이미지·한글 파일 불가)
• 서류별로 파일을 개별 업로드 (하나로 합쳐서 올리면 안 됨)
• 접수번호는 문자나 메모 앱에 저장해 두세요
3
관리자 화면 안내
admin.html · 로그인: admin / 1234
①
접수 현황
회차별 접수 목록을 확인합니다. 이름·학교 검색, 구분·상태 필터 사용 가능.
완비 일부 누락 미완료 상태로 서류 현황을 한눈에 파악할 수 있습니다.
⚠ 표시가 있는 학생은 중복 수혜 또는 가족 중복 가능성이 있으니 확인하세요.
수동 입력 버튼으로 방문 접수 학생을 직접 등록할 수 있습니다.
②
개인 상세 패널 (목록에서 이름 클릭)
• 서류 제출 현황 확인 및 관리자 직접 PDF 업로드
• 누락 서류 안내 메시지 자동 생성 → 복사 후 이메일·카카오 발송
• 발송 완료 표시 버튼으로 안내 발송 이력 기록
• 선발 여부 체크 (선발 / 미선발)
• 특이사항 메모 입력 (내용이 누적 저장됩니다)
③
선발 현황
• 회차별 구분별 접수 인원 / 선발 인원 / 선발률 테이블
• 동일인 중복 수혜 자동 감지 (이름 + 생년월일 기준)
• 가족 중복 수혜 자동 감지 (동일 주소, 다른 이름)
• 이름 검색으로 현재 + 이전 이력 통합 조회
• 선발 명단 CSV 파일 내보내기
④
이전 이력
과거 선발 데이터를 CSV로 등록합니다. 선발 현황의 중복 수혜 검토에 자동으로 반영됩니다. 아래 4번 항목을 참고하세요.
⑤
설정
접수 회차와 구분을 추가·편집합니다. 각 구분별 접수 기간과 서류 목록을 자유롭게 설정할 수 있습니다. 아래 5번 항목을 참고하세요.
CSV 백업 방법
1
접수 현황 탭 → CSV 버튼 클릭
현재 선택된 회차의 전체 접수 데이터가 엑셀 호환 CSV 파일로 저장됩니다.
2
접수 기간 종료 후 파일(PDF) 별도 보관
심사 완료 후 업로드된 PDF 파일을 백업 저장하고 시스템에서 삭제하면 저장 공간이 초기화됩니다.
4
이전 이력 등록
과거 선발 데이터를 CSV로 업로드하여 중복 수혜를 자동 검토
등록 절차
1
이전 이력 탭 → CSV 양식 다운로드
버튼 클릭 시 양식 파일이 자동으로 저장됩니다.
2
양식에 이전 이력 입력
엑셀에서 양식을 열어 데이터를 입력합니다. 고등부는 생년월일 칸을 비워도 됩니다.
3
CSV로 저장 후 업로드
'다른 이름으로 저장 → CSV(쉼표로 구분)' 형식으로 저장 후 업로드 버튼으로 불러옵니다.
컬럼명입력 예시비고
이름홍길동필수
생년월일990101고등부는 빈칸 가능
주소경기도 의왕시 포일로 123도로명주소 권장
상세주소101동 505호선택
회차명2025년 상반기자유롭게 입력
구분명대학부 희망드림자유롭게 입력
선발여부선발선발 또는 미선발
현재 시스템에는 2025년 예시 이력 10건이 기본으로 입력되어 있습니다.
실제 운영 시 초기화 버튼으로 삭제 후 실제 데이터를 입력하세요.
5
설정 방법
admin.html → 설정 탭
회차 추가 (예: 2027년 상반기)
1
회차 추가 버튼 클릭
회차명 입력 후 저장 (예: 2027년 상반기)
2
구분 추가 버튼 클릭
분야별로 구분을 추가합니다 (예: 대학부 희망드림, 고등부 성적우수 등)
3
구분별 세부 설정
접수 시작일·종료일·마감시간 설정, 서류 목록 항목 추가 및 필수/선택 여부 지정
접수 기간 자동 관리
설정한 기간이 되면 학생 페이지에서 자동으로 해당 분야가 활성화됩니다. 기간이 지나면 자동으로 잠깁니다. 별도 조작이 필요 없습니다.
관리자 비밀번호 변경
admin.html 파일을 메모장 또는 텍스트 편집기로 열어
const ACCOUNTS=[{id:'admin',pw:'1234',name:'담당자'}];
이 부분에서 pw 값을 원하는 비밀번호로 변경하면 됩니다.
6
자주 묻는 질문
운영 중 자주 발생하는 상황 대처 방법
접수가 열려 있어야 하는데 학생 화면에서 잠겨 있어요
설정 탭 → 해당 구분 편집 → 접수 시작일과 종료일을 확인하세요. 날짜 형식이 정확한지 확인 후 저장합니다.
학생이 파일을 잘못 올렸어요
관리자 접수 현황 → 해당 학생 클릭 → 상세 패널에서 해당 서류 옆 [직접 업로드] 버튼으로 올바른 파일을 덮어쓸 수 있습니다.
학생이 접수번호를 잃어버렸어요
관리자 목록에서 이름 또는 학교명으로 검색합니다. 개인 상세 패널 상단에서 접수번호를 확인하여 학생에게 안내합니다.
같은 학생이 실수로 두 번 접수했어요
중복 접수된 건의 특이사항 메모에 "중복 접수 - 무효 처리" 등을 기록하고 별도 관리합니다. 선발 여부를 미선발로 처리하면 선발 현황에서 제외됩니다.
방문 접수 학생은 어떻게 등록하나요?
접수 현황 탭 → [+ 수동 입력] 버튼 클릭 → 학생 정보와 서류 제출 현황을 체크하여 저장합니다. 목록에 "수동" 태그로 구분 표시됩니다.
다른 컴퓨터에서 접속하면 데이터가 없어요
현재는 브라우저 로컬 저장소를 사용하기 때문에 같은 기기·같은 브라우저에서만 데이터가 유지됩니다. 다른 기기에서도 공유하려면 Firebase 연동이 필요합니다. 담당자에게 문의하세요.
